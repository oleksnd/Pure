# PROJECT PURE: TECHNICAL MANIFESTO

## 1. QUALITY & ARCHITECTURE
- **Clean Code:** Применяй принципы DRY и KISS. Код должен быть модульным и читаемым.
- **File Structure:** Нет жесткого ограничения на один файл, но каждое разделение должно быть обосновано удобством поддержки.
- **No Placeholders:** Запрещено использовать комментарии-заглушки (напр. "// logic goes here"). Пиши полный рабочий код.
- **No Bloat:** Запрещено подключать сторонние библиотеки (React, jQuery, Tailwind и др.) или внешние скрипты через CDN без прямого разрешения. Используй Vanilla JS и чистый CSS.

## 2. GEMINI NANO INTEGRATION (BUILT-IN AI)
Ты — эксперт в Web AI. Реализация функций ИИ должна следовать спецификациям Chrome Prompt API:
- **Capability Check:** Всегда проверяй `(await ai.languageModel.capabilities()).available`.
- **Session Management:** Используй `ai.languageModel.create()` с обработкой ошибок.
- **Streaming:** По возможности используй `promptStreaming()` для лучшего UX.
- **Graceful Degradation:** Если ИИ недоступен в браузере, интерфейс должен предлагать альтернативу или скрывать ИИ-функции, не ломая основную логику.

## 3. SECURITY & INTEGRITY
- **Data Safety:** Не допускай утечек данных в консоль или внешние запросы.
- **Zero Hallucinations:** Если метод API тебе неизвестен или нестабилен — сообщи об этом. Не выдумывай синтаксис.
- **Audit:** Твой код проверяется контролирующим Ассистентом. Попытки скрыть баги или выдать нерабочий код за финальный будут пресечены.

## 4. COMMUNICATION WITH OWNER
- Владелец проекта — не программист.
- После каждого изменения давай краткий отчет: "Что сделано" и "Как это проверить" простыми словами.
- Если предлагаешь рефакторинг — объясни выгоду (скорость, память, чистота).
